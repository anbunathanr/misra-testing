# Direct API Test for Bedrock - No UI Required

**Status**: Ready to Execute  
**Date**: April 1, 2026

## Quick Summary

The AI Test Generation feature is accessed via a direct API endpoint, not through a UI page. You can test it directly using PowerShell.

## API Endpoint

```
POST https://aibts-platform.vercel.app/api/ai-test-generation/generate
```

## Step 1: Get Your Auth Token

First, you need to log in to get an authentication token:

```powershell
# Login to get token
$loginUrl = "https://aibts-platform.vercel.app/api/auth/login"
$loginBody = @{
    email = "your-email@example.com"
    password = "your-password"
} | ConvertTo-Json

$loginResponse = Invoke-WebRequest -Uri $loginUrl -Method POST -Body $loginBody -ContentType "application/json"
$token = ($loginResponse.Content | ConvertFrom-Json).token
Write-Host "Token: $token"
```

## Step 2: Test Bedrock via API

Once you have the token, make a test generation request:

```powershell
# Set your token
$token = "YOUR_TOKEN_HERE"

# Prepare the request
$url = "https://aibts-platform.vercel.app/api/ai-test-generation/generate"

$body = @{
    analysis = @{
        url = "https://example.com/login"
        elements = @(
            @{
                selector = "input[name='email']"
                type = "input"
                label = "Email"
            },
            @{
                selector = "input[name='password']"
                type = "input"
                label = "Password"
            },
            @{
                selector = "button[type='submit']"
                type = "button"
                label = "Login"
            }
        )
    }
    scenario = "User logs in with valid credentials"
    projectId = "test-project-123"
    suiteId = "test-suite-456"
} | ConvertTo-Json -Depth 10

# Make the request
$headers = @{
    "Authorization" = "Bearer $token"
    "Content-Type" = "application/json"
}

$response = Invoke-WebRequest -Uri $url -Method POST -Body $body -Headers $headers
$result = $response.Content | ConvertFrom-Json

Write-Host "Response Status: $($response.StatusCode)"
Write-Host "Test Generated: $($result.testCase.name)"
Write-Host "Cost: $($result.cost)"
```

## Step 3: Monitor CloudWatch Logs

While the API request is processing, monitor the Lambda logs:

```powershell
aws logs tail /aws/lambda/aibts-ai-generate --follow
```

## What to Look For in Logs

You should see:
```
[INFO] Creating AI engine for provider: BEDROCK
[INFO] Invoking Bedrock model: us.anthropic.claude-sonnet-4-6
[INFO] Bedrock response received
[INFO] Generated test code: ...
```

## Alternative: Simple Test with curl

If you prefer curl:

```bash
curl -X POST https://aibts-platform.vercel.app/api/ai-test-generation/generate \
  -H "Authorization: Bearer YOUR_TOKEN_HERE" \
  -H "Content-Type: application/json" \
  -d '{
    "analysis": {
      "url": "https://example.com/login",
      "elements": [
        {"selector": "input[name=email]", "type": "input", "label": "Email"},
        {"selector": "input[name=password]", "type": "input", "label": "Password"},
        {"selector": "button[type=submit]", "type": "button", "label": "Login"}
      ]
    },
    "scenario": "User logs in with valid credentials",
    "projectId": "test-project-123",
    "suiteId": "test-suite-456"
  }'
```

## Success Indicators

✅ Response status code: 200  
✅ Response contains `testCase` object  
✅ Response contains `cost` value  
✅ CloudWatch logs show Bedrock being used  
✅ Test code is generated by Claude Sonnet 4.6  

## Troubleshooting

**401 Unauthorized**
- Token is invalid or expired
- Get a new token by logging in again

**400 Bad Request**
- Missing required fields (analysis, scenario, projectId, suiteId)
- Check the request body format

**429 Too Many Requests**
- Rate limit hit
- Wait 10-15 minutes before retrying

**500 Internal Server Error**
- Check CloudWatch logs for details
- May be Bedrock configuration issue
