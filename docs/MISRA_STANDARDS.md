## Official MISRA Standards

### MISRA Organization

**MISRA** (Motor Industry Software Reliability Association) is a collaboration between vehicle manufacturers, component suppliers, and engineering consultancies that develops software development guidelines for the automotive industry.

**Official Website**: [https://www.misra.org.uk](https://www.misra.org.uk)

**Contact Information**:
- Email: info@misra.org.uk
- Address: MISRA, HORIBA MIRA Ltd, Watling Street, Nuneaton, Warwickshire, CV10 0TU, United Kingdom

### About MISRA Guidelines

MISRA guidelines are:
- **Industry Standards**: Widely adopted in automotive, aerospace, medical, and other safety-critical industries
- **Language-Specific**: Separate guidelines for C and C++
- **Rule-Based**: Provide specific, checkable rules for code quality
- **Safety-Focused**: Designed to avoid undefined behavior and improve reliability
- **Commercially Available**: Must be purchased from MISRA or authorized distributors

### Standard Contents

The MISRA C:2012 standard includes:

1. **Introduction**

   - Advisory rules (12)

3. **Rule Categories**:
   - Standard environment
   - Language extensions
   - Comments
   - Character sets
   - Identifiers
   - Types
   - Literals and constants
   - Declarations and definitions
   - Initialization
   - Arithmetic type conversions
   - Pointer type conversions
   - Expressions
   - Side effects
   - Control statement expressions
   - Control flow
   - Switch statements
   - Functions
   - Pointers and arrays
   - Overlapping storage
   - Preprocessing directives
   - Standard libraries
   - Resources
   - Error handling

4. **Appendices**:
   - Decidability of rules
   - Implementation compliance
   - Rule classification
   - Summary of guidelines

### Key Updates from MISRA C:2004

- Aligned with C99 standard (ISO/IEC 9899:1999)
- Reorganized rule structure
- Added decidability classification
- Improved rule clarity
- Added new rules for modern C features

### Amendment 1 (2016)

**Title**: MISRA C:2012 Amendment 1 - Additional security guidelines for MISRA C:2012

**Publication Date**: April 2016

**Focus**: Security-related rules and guidelines

**Purchase**: Available separately or bundled with main standard

MISRA C++:2008 is based on:
- **ISO/IEC 14882:2003** (C++03 standard)
- Compatible with C++98
- Some rules applicable to C++11 and later

**MISRA C++:202x** (in development):
- Will align with modern C++ standards (C++11/14/17)
- Expected to include new rules for modern C++ features
- Release date to be announced

## Related Standards

### ISO C Standards

**ISO/IEC 9899:2011 (C11)**
- Official C language standard
- Basis for MISRA C:2012
- Available from: [https://www.iso.org](https://www.iso.org)

**ISO/IEC 9899:2018 (C17/C18)**
- Latest C standard
- Minor update to C11
- MISRA C:2012 largely compatible

### ISO C++ Standards

**ISO/IEC 14882:2003 (C++03)**
- Basis for MISRA C++:2008
- Available from: [https://www.iso.org](https://www.iso.org)

**ISO/IEC 14882:2011 (C++11)**
- Modern C++ standard
- Not yet covered by MISRA C++

**ISO/IEC 14882:2014 (C++14)**
- Refinement of C++11

**ISO/IEC 14882:2017 (C++17)**
- Latest major C++ standard

### Safety Standards

**ISO 26262** - Road vehicles - Functional safety
- Automotive functional safety standard
- References MISRA C/C++ for software development
- Website: [https://www.iso.org/standard/68383.html](https://www.iso.org/standard/68383.html)

**IEC 61508** - Functional safety of electrical/electronic/programmable electronic safety-related systems
- Generic functional safety standard
- Applicable across industries
- Website: [https://www.iec.ch](https://www.iec.ch)

**DO-178C** - Software Considerations in Airborne Systems and Equipment Certification
- Aerospace software standard
- Often used with MISRA C/C++
- Website: [https://www.rtca.org](https://www.rtca.org)

**IEC 62304** - Medical device software lifecycle processes
- Medical device software standard
- References coding standards like MISRA
- Website: [https://www.iec.ch](https://www.iec.ch)

## Compliance and Certification

### Achieving MISRA Compliance

**Steps to Compliance**:
1. **Adopt Standard**: Purchase and study MISRA guidelines
2. **Configure Tools**: Set up static analysis tools
3. **Train Team**: Educate developers on rules
4. **Establish Process**: Define compliance procedures
5. **Document Deviations**: Create deviation process
6. **Verify Compliance**: Use tools and reviews
7. **Maintain Records**: Keep compliance documentation

### Deviation Procedures

**When Deviations Are Allowed**:
- Technical impossibility
- Performance requirements
- Legacy code constraints
- Tool limitations

**Deviation Documentation**:
- Rule being deviated
- Justification
- Alternative measures
- Approval signatures
- Review date

### Compliance Matrix

A compliance matrix should include:
- List of all applicable rules
- Compliance status (Pass/Fail/Deviated)
- Deviation justifications
- Tool coverage
- Manual review results

## Frequently Asked Questions

### General Questions

**Q: Do I need to purchase MISRA standards?**
A: Yes, MISRA standards are commercially available and must be purchased from MISRA or authorized distributors.

**Q: Can I use MISRA for non-automotive projects?**
A: Yes, MISRA is applicable to any safety-critical or high-reliability software project.

**Q: Is MISRA compliance mandatory?**
A: It depends on your industry and contracts. Many automotive and safety-critical projects require MISRA compliance.

**Q: How often are MISRA standards updated?**
A: Major updates occur every 5-10 years, with amendments and clarifications released as needed.

### Technical Questions

**Q: Can I deviate from MISRA rules?**
A: Yes, with proper justification and documentation. See MISRA Compliance:2020 for guidance.

**Q: Which MISRA standard should I use?**
A: Use MISRA C:2012 for C projects and MISRA C++:2008 for C++ projects. Consider your target language standard.

**Q: Are all MISRA rules checkable by tools?**
A: No, some rules require manual review. Check the decidability classification in the standard.

**Q: Can I use C++11/14/17 with MISRA C++:2008?**
A: MISRA C++:2008 is based on C++03. Many rules still apply, but modern features aren't covered. Wait for MISRA C++:202x.
